HTML Learning Roadmap

1. HTML Fundamentals (Building Blocks 🧱)

HTML (HyperText Markup Language) is the foundational language used to create web pages. It defines the structure and layout of a website by using various "elements" and "tags" to organize content. Understanding the fundamentals of HTML is essential to becoming proficient in web development.

1.1 Introduction to HTML:

Definition:
HTML is the standard language for creating and designing webpages. It provides the structure for content such as text, images, links, and multimedia. HTML documents are read by web browsers, which interpret the code to display the content.

Role in Web Development:
HTML acts as the skeleton of a webpage. It organizes content into readable sections for browsers, enabling users to interact with the webpage. HTML works with other technologies like CSS (Cascading Style Sheets) for styling and JavaScript for interactivity.

Example:

<!DOCTYPE html>
<html>
<head>
	<title>My First Webpage</title>
</head>
<body>
	<h1>Welcome to My Website!</h1>
	<p>This is a simple HTML page.</p>
</body>
</html>

Output:
When this code is opened in a browser, it will display:

Welcome to My Website!

This is a simple HTML page.

1.2 HTML Editors:

Definition:
HTML editors are software tools designed to help developers write and edit HTML code. They often come with features like syntax highlighting, auto-completion, and error checking, making the coding process easier.

Types of HTML Editors:

  • Text Editors: Simple text editors like Notepad (Windows) or TextEdit (Mac) can be used to write HTML, though they lack advanced features.
  • Code Editors: These include Visual Studio Code (VSCode), Sublime Text, Atom, and Brackets. They provide features like syntax highlighting, error detection, and extensions.
  • Integrated Development Environments (IDEs): Full-featured development environments like WebStorm and Eclipse that support HTML, CSS, and JavaScript.

Example of Code Editor:
A developer may use Visual Studio Code (VSCode) to write HTML with features like live preview, code snippets, and auto-completion for better productivity.

1.3 Basic Structure:

Definition:
The basic structure of an HTML document consists of the document declaration, <html> element, <head> section (metadata), and the <body> section (actual content visible on the page).

Structure Breakdown:

  • <!DOCTYPE html>: Declares the document type, which is HTML5 in this case.
  • <html>: The root element that contains all other HTML elements.
  • <head>: Contains meta-information about the document, such as title, character encoding, and links to stylesheets or scripts.
  • <body>: Contains the content of the webpage, such as text, images, and links.

Example:

<!DOCTYPE html>
<html>
<head>
	<meta charset="UTF-8">
	<title>Sample Webpage</title>
</head>
<body>
	<h1>HTML Basic Structure</h1>
	<p>This is the body of the webpage.</p>
</body>
</html>

Output:
The webpage will display:

HTML Basic Structure

This is the body of the webpage.

1.4 Elements and Attributes:

Definition:

Elements: HTML elements are the building blocks of an HTML page. They define different types of content, such as headings, paragraphs, images, and links. Each element is typically enclosed by opening and closing tags.

Example: <h1>Welcome</h1>, <p>This is a paragraph.</p>

Attributes: Attributes provide additional information about HTML elements. They appear inside the opening tag and are written as a key-value pair (e.g., src="image.jpg" for an image). Attributes are used to customize the behavior or appearance of an element.

Common Attributes:

  • id: Specifies a unique identifier for an element.
  • class: Specifies one or more class names for an element.
  • href: Specifies the destination URL for a hyperlink.
  • src: Specifies the source of an image.
  • alt: Provides an alternative text for an image.

Example:

<!DOCTYPE html>
<html>
<head>
	<title>HTML Elements and Attributes</title>
</head>
<body>
	<h1 id="main-header">HTML Example</h1>
	<p class="intro">This is a paragraph with a class attribute.</p>
	<a href="https://www.example.com">Click here</a> to visit Example.com.
	<img src="image.jpg" alt="A sample image">
</body>
</html>

Explanation of the elements:

  • <h1 id="main-header">: This is a heading element with an id attribute set to "main-header".
  • <p class="intro">: This is a paragraph with a class attribute set to "intro".
  • <a href="https://www.example.com">: This is a hyperlink with the href attribute pointing to "https://www.example.com".
  • <img src="image.jpg" alt="A sample image">: This is an image element with src pointing to "image.jpg" and alt providing a description of the image.

Output:
A webpage with a heading "HTML Example."

HTML Example

This is a paragraph with a class attribute.

Click here to visit Example.com.

2. Text Content and Styling (✍️ Writing with Flair)

Text is the core of most web pages. In this module, we’ll explore the various ways to structure and style text content, creating web pages that are both readable and visually appealing.

2.1 Headings:

Definition:
Headings are used to define the structure of a webpage. They provide a clear hierarchy for the content, making it easier to navigate and read.

Heading Levels:
HTML provides six levels of headings, from <h1> (most important) to <h6> (least important).

Example:


<h1>Main Title</h1>
<h2>Section Title</h2>
<h3>Subsection Title</h3>
<h4>Subsection Title</h4>
<h5>Subsection Title</h5>
<h6>Subsection Title</h6>

Output:
A webpage with headings organized by size and importance:

Main Title

Section Title

Subsection Title

Subsection Title

Subsection Title
Subsection Title

2.2 Paragraphs:

Definition:
Paragraphs are used to group related sentences or text content into distinct blocks for readability. The <p> element is used to create paragraphs.

Example:


<p>This is the first paragraph.</p>
<p>This is the second paragraph with more text.</p>

Output:
Two paragraphs displayed on a webpage:

This is the first paragraph.

This is the second paragraph with more text.

2.3 Styles:

Definition:
Styling text enhances its appearance on a webpage. While CSS (covered in a later module) is the primary styling tool, basic styles can be applied using HTML attributes and tags.

Basic Styling Options:

  • <b> and <strong>: Bold text.
  • <i> and <em>: Italicize text.
  • <u>: Underline text.

Example:


<p>This is <b>bold</b>, <i>italicized</i>, and <u>underlined</u> text.</p>

Output:
A paragraph with styled text:

This is bold, italicized, and underlined text.

2.4 Formatting:

Definition:
Formatting refers to enhancing text readability and presentation by adding emphasis, quotes, and comments. It helps organize content and improve code clarity.

Common Formatting Options:

  • <blockquote>: Used for long quotations.
  • <abbr>: Defines abbreviations.
  • <mark>: Highlights text.
  • HTML comments: Use <!-- comment --> to add notes that are not displayed on the webpage.

Example:


<blockquote>
This is a long quote from a famous author.
</blockquote>

<abbr title="HyperText Markup Language">HTML</abbr> is a foundational 
web technology.

<mark>Important text</mark> is highlighted.

<!-- This is a comment and will not appear in the browser -->

Output:
A webpage with formatted content:

This is a long quote from a famous author.

HTML is a foundational web technology.

Important text is highlighted.

3. Colors, Links, and Images (🎨 Visual Enhancements)

Enhancing the visual and navigational elements of a webpage is crucial in creating an engaging user experience. In this module, we explore how to add colors, create hyperlinks, embed images, include favicons, and define descriptive page titles.

3.1 Colors:

Definition:
Colors in HTML help highlight text, set background themes, and improve the visual appeal of a webpage. They can be specified using predefined color names, hexadecimal (HEX) codes, or RGB values.

Methods of Adding Colors:

  • Color Names: Predefined names like "red", "blue", and "green".
  • HEX Codes: Represent colors using hexadecimal values, e.g., #FF0000 for red.
  • RGB: Specify colors based on the intensity of red, green, and blue (0-255), e.g., rgb(255, 0, 0) for red.

Example:

<html>
	<body>
		<p style="color:green;">This text is green.</p>
		<p style="color:#FF5733;">This text uses a HEX color.</p>
		<p style="color:rgb(0, 0, 255);">This text uses an RGB color.</p>
	</body>
</html>

Output:

This text is green.

This text uses a HEX color.

This text uses an RGB color.

3.2 Links:

Definition:
Links allow navigation between web pages or sections of the same page. The <a> element is used to define a hyperlink, with the href attribute specifying the destination.

Attributes:

  • href: Specifies the URL of the page to navigate to.
  • target: Specifies where to open the linked document (e.g., in the same tab or a new tab).
  • title: Provides additional information about the link when hovered.

Example:


<html>
<body>
	<a href="https://www.example.com">Visit Example Website</a>
	<a href="#section1">Jump to Section 1 on the same page</a>
</body>
</html>

Output:

Visit Example Website
Jump to Section 1 on the same page

3.3 Images:

Definition:
Images add a visual element to web pages, helping convey information or enhance appeal. The <img> tag is used to embed images into a webpage.

Attributes:

  • src: Specifies the file path or URL of the image.
  • alt: Provides alternative text displayed if the image fails to load or for accessibility.
  • width and height: Define the dimensions of the image (in pixels).

Example:


<html>
<body>
	<img src="example.jpg" alt="A sample image" width="300" height="200">
</body>
</html>

Output:

3.4 Favicon:

Definition:
A favicon is a small icon displayed in the browser tab and bookmarks. It represents the website and enhances brand identity.

How to Add a Favicon:
Include the <link> element in the <head> section, specifying the favicon file path and type.

Example:


<html>
	<head>
		<link rel="icon" href="favicon.ico" type="image/x-icon">
	</head>
	<body>
		<p>This page has a favicon.</p>
	</body>
</html>

Output:

This page has a favicon.

3.5 Page Title:

Definition:
The page title describes the content of the webpage and is displayed on the browser tab and search engine results. It is defined using the <title> element inside the <head> section.

Example:

<html>
	<head>
		<title>My First Webpage</title>
	</head>
	<body>
		<p>Welcome to my webpage!</p>
	</body>
</html>

Output:

Welcome to my webpage!

4. Tables and Lists

Tables and lists are essential tools for organizing and presenting information on web pages. In this module, we will explore how to create structured tables, ordered and unordered lists, datalists, and advanced table formatting using colspan and rowspan.

4.1 Tables:

Definition:
A table is a structured format for displaying data in rows and columns. HTML tables are created using the <table> tag, with rows defined by the <tr> tag and cells within rows defined by the <td> (for regular cells) and <th> (for header cells) tags.

Key Tags in Tables:

  • <table>: Used to define the table structure.
  • <tr>: Defines a row in the table.
  • <th>: Defines a header cell, typically bold and centered.
  • <td>: Defines a regular data cell.
  • colspan: Merges two or more columns into one cell.
  • rowspan: Merges two or more rows into one cell.

Example:

<html>
	<body>
		<table border="1">
			<tr>
				<th rowspan="2">Product</th>
				<th colspan="2">Details</th>
			</tr>
			<tr>
				<th>Price</th>
				<th>Stock</th>
			</tr>
			<tr>
				<td>Apples</td>
				<td>$2</td>
				<td>50</td>
			</tr>
			<tr>
				<td>Bananas</td>
				<td>$1</td>
				<td>100</td>
			</tr>
		</table>
	</body>
</html>

Output:

Product Details
Price Stock
Apples $2 50
Bananas $1 100

4.2 Lists:

Definition:
Lists are used to present a collection of related items in a structured format. HTML supports two main types of lists: unordered and ordered, and an additional definition list type for terms and definitions.

4.2.1 Unordered Lists:

Unordered lists display items with bullet points. They are created using the <ul> tag, and each item within the list is defined using the <li> tag.

Example:

<html>
	<body>
		<ul>
			<li>HTML</li>
			<li>CSS</li>
			<li>JavaScript</li>
		</ul>
	</body>
</html>

Output:

  • HTML
  • CSS
  • JavaScript

4.2.2 Ordered Lists:

Ordered lists display items in a specific sequence, using numbers or letters. They are created using the <ol> tag, with items defined using the <li> tag.

Example:

<html>
	<body>
		<ol>
			<li>Step 1: Open the editor.</li>
			<li>Step 2: Write HTML code.</li>
			<li>Step 3: Save and preview.</li>
		</ol>
	</body>
</html>

Output:

  1. Step 1: Open the editor.
  2. Step 2: Write HTML code.
  3. Step 3: Save and preview.

4.2.3 Definition Lists:

Definition lists are used to pair terms with their definitions. They are created using the <dl> tag, with terms defined by <dt> and their descriptions by <dd>.

Example:

<html>
	<body>
		<dl>
			<dt>HTML</dt>
			<dd>A markup language for creating web pages.</dd>
			<dt>CSS</dt>
			<dd>A stylesheet language for designing web pages.</dd>
		</dl>
	</body>
</html>

Output:

HTML
A markup language for creating web pages.
CSS
A stylesheet language for designing web pages.

4.3 Datalist:

Definition:
A datalist provides a dropdown of predefined options for an input field, enhancing user experience by allowing suggestions.

Key Tags in Datalist:

  • <datalist>: Contains the list of options.
  • <option>: Represents individual options in the datalist.

Example:

<html>
	<body>
		<label for="browser">Choose a browser:</label>
		<input list="browsers" id="browser" name="browser">
		<datalist id="browsers">
			<option value="Chrome">
			<option value="Firefox">
			<option value="Safari">
			<option value="Edge">
		</datalist>
	</body>
</html>

Output:

5. Structure

Understanding the structural elements of HTML is key to building organized and maintainable web pages. This module focuses on block and inline elements, the versatile <div> tag, and how to use classes and IDs, file paths, iframes, and the <head> section effectively.

5.1 Block and Inline Elements:

Definition:
Block elements occupy the full width available and always start on a new line. Examples include <div>, <p>, and <h1>. Inline elements, on the other hand, do not start on a new line and only take up as much width as necessary, such as <span>, <a>, and <strong>.

Example:

<html>
	<body>
		<div>This is a block element.</div>
		<span>This is an inline element.</span>
	</body>
	</html>

Output:

This is a block element.
This is an inline element.

5.2 Div Tag:

Definition:
The <div> tag is a container for grouping elements together and applying styles or scripts. It is a block-level element commonly used for layout and structure.

Example:

<html>
	<body>
		<div class="contain">
			<h1>Welcome</h1>
			<p>This is a paragraph inside a div.</p>
		</div>
	</body>
	</html>

Output:

Welcome

This is a paragraph inside a div.

5.3 Classes and IDs:

Definition:
Classes and IDs are attributes used to apply styles or scripts to specific HTML elements. A class can be applied to multiple elements, while an ID must be unique within a page.

Example:

<html>
	<body>
		<div class="example">This is a div with a class.</div>
		<div id="unique">This is a div with an ID.</div>
	</body>
	</html>

Output:

This is a div with a class.
This is a div with an ID.

5.4 File Paths:

Definition:
File paths specify the location of a file to be used in an HTML document. Paths can be relative (based on the current document's location) or absolute (full URL or root-based).

Example:

<html>
	<body>
		<img src="images/logo.png" alt="Logo">
		<a href="/about.html">About Us</a>
	</body>
	</html>

Output:

About Us

5.5 Iframes:

Definition:
The <iframe> tag embeds another HTML document within the current document.

Example:

<html>
	<body>
		<iframe src="https://example.com" width="600" height="400"></iframe>
	</body>
	</html>

Output:

5.6 Head Section:

Definition:
The <head> section contains metadata and references to resources for the web page, such as CSS, JavaScript, and SEO-related tags. It does not display content directly but is crucial for functionality and optimization.

Common Tags in the Head Section:

  • <title>: Specifies the title of the web page.
  • <meta>: Provides metadata like character encoding, viewport settings, and description.
  • <link>: Links external resources like stylesheets.
  • <script>: Links JavaScript files or includes inline JavaScript.
  • <style>: Contains inline CSS rules for styling.

Example:

<html>
	<head>
		<title>My Web Page</title>
		<meta charset="UTF-8">
		<meta name="viewport" content="width=device-width, initial-scale=1.0">
		<link rel="stylesheet" href="styles.css">
		<script src="script.js"></script>
		<style>
			body {
				font-family: Arial, sans-serif;
				background-color: #f4f4f4;
			}
		</style>
	</head>
	<body>
		<h1>Welcome to My Page</h1>
	</body>
	</html>

Output:

Welcome to My Page

5.7 HTML Layout Elements

HTML has several semantic elements that define the different parts of a web page. These elements help structure content in a meaningful way, making it easier to maintain and optimize.

5.7.1 <header>

Definition:
The <header> element represents the introductory content or navigation links of a section or the entire document.

Example:

<header>
	<h1>Welcome to My Website</h1>
	<nav>
		<a href="#home">Home</a> |
		<a href="#about">About Us</a> |
		<a href="#services">Services</a>
	</nav>
</header>

Output:

Welcome to My Website

Home | About Us | Services

5.7.2 <nav>

Definition:
The <nav> element defines a set of navigation links. It groups links together to form a navigation menu.

Example:

<nav>
	<a href="#home">Home</a>
	<a href="#contact">Contact</a>
</nav>

Output:

Home Contact

5.7.3 <section>

Definition:
The <section> element defines a section of content. It groups related content together, such as a chapter in a book or a blog post.

Example:

<section>
	<h2>Our Services</h2>
	<p>We offer a wide range of web development services.</p>
</section>

Output:

Our Services

We offer a wide range of web development services.

5.7.4 <article>

Definition:
The <article> element represents a self-contained piece of content that could be distributed and reused, like a news article or blog post.

Example:

<article>
	<h3>Latest News</h3>
	<p>Here is the latest update on our services.</p>
</article>

Output:

Latest News

Here is the latest update on our services.

5.7.5 <aside>

Definition:
The <aside> element represents content that is tangentially related to the content around it, often used for sidebars, pull quotes, or related links.

Example:

<aside>
	<h4>Related Links</h4>
	<ul>
		<li><a href="#service1">Service 1</a></li>
		<li><a href="#service2">Service 2</a></li>
	</ul>
</aside>

Output:

Related Links

  • Service 1
  • Service 2

5.7.6 <footer>

Definition:
The <footer> element defines a footer for a document or section, typically containing copyright information, contact links, or additional information.

Example:

<footer>
	<p>Copyright © 2024 My Website</p>
</footer>

Output:

Copyright © 2024 My Website

5.7.7 <details>

Definition:
The <details> element defines additional details that the user can open or close on demand. Typically used for FAQs or expandable content.

Example:

<details>
	<summary>Click to see more details</summary>
	<p>Here are some additional details that are hidden by default.</p>
</details>

Output:

Click to see more details

Here are some additional details that are hidden by default.

5.7.8 <summary>

Definition:
The <summary> element defines a heading for the <details> element. It acts as the clickable element that expands or collapses the hidden content.

Example:

<details>
	<summary>More Info</summary>
	<p>This is some additional content that is hidden by default.</p>
</details>

Output:

More Info

This is some additional content that is hidden by default.

6.1 Introduction to Semantic Elements

In HTML5, semantic elements are those that clearly describe their meaning in a human- and machine-readable way. These elements are used to structure the content of a webpage in a meaningful way, enhancing the readability and maintainability of the code. Semantic elements help describe the role of the content inside them and make web pages easier to navigate for both users and search engines.

Definition:
Semantic elements are HTML tags that carry meaning about their content. For example, elements like <header>, <article>, <section>, and <footer> describe parts of a webpage that have specific meaning. This helps with SEO and accessibility, as these elements provide a clear structure that search engines and screen readers can interpret.

Example:


<header>
	<h1>My Webpage</h1>
	<nav>Navigation links here</nav>
</header>
<article>
	<h2>Article Title</h2>
	<p>Content of the article here</p>
</article>

Output:

My Webpage

Navigation links here

Article Title

Content of the article here

6.2 Importance of Semantics

Using semantic elements in HTML is crucial for both accessibility and SEO. These elements provide context to the content, helping search engines to understand the structure of the webpage better. They also help with accessibility, making it easier for screen readers to convey the content of the page to users with disabilities.

6.2.1 Advantages of Semantic Elements

  • SEO (Search Engine Optimization): Search engines like Google give more value to well-structured, semantic content. Using semantic elements helps improve the ranking of your webpage in search results.
  • Improved Accessibility: Semantic tags improve the experience for people who use screen readers or other assistive technologies. It helps these devices understand the content and its purpose.
  • Readability and Maintenance: Semantic elements make it easier for developers to read and maintain code. It's clear to anyone reading the HTML what each part of the page represents.
  • Better Code Organization: Semantic elements provide a clearer organization of the page layout and content structure, making it easier to manage.

Example: A simple layout using semantic elements for better structure:


<header>
	<h1>Welcome to My Website</h1>
	<nav>Links to sections</nav>
</header>
<main>
	<article>
		<h2>Article Title</h2>
		<p>Content of the article</p>
	</article>
	<section>
		<h2>Another Section</h2>
		<p>Content of the section</p>
	</section>
</main>
<footer>
	<p>Copyright © 2024 My Website</p>
</footer>

Output:

Welcome to My Website

Links to sections

Article Title

Content of the article

Another Section

Content of the section

Copyright © 2024 My Website

6.3 What Are Semantic Elements?

Definition:
A semantic element clearly describes its meaning to both the browser and the developer. It tells both the browser and the developer the content's role within the document.

Examples of Non-Semantic Elements:
Non-semantic elements, such as <div> and <span>, tell nothing about their content and are used purely for styling or scripting purposes.

Examples of Semantic Elements:
Semantic elements, like <article>, <form>, and <header>, clearly define their content and are meaningful for both developers and search engines.

<article>

Definition: The <article> element represents a self-contained piece of content that could be distributed and reused independently. An article should make sense on its own and be part of a larger site or context.

Example:


<article>
	<h2>How to Learn HTML</h2>
	<p>HTML is the foundation of web development...</p>
</article>

<aside>

Definition: The <aside> element defines content that is tangentially related to the content around it. It is typically used for sidebars, pull quotes, or related links.

Example:


<aside>
	<h3>Related Articles</h3>
	<p>Learn more about web design...</p>
</aside>

<details>

Definition: The <details> element represents a disclosure widget from which the user can obtain additional information or controls. It typically contains a <summary> tag to define a heading.

Example:


<details>
	<summary>More Information</summary>
	<p>Here is the additional information you requested.</p>
</details>

<figcaption>

Definition: The <figcaption> element is used to provide a caption or description for a <figure>, which could be an image, diagram, or other media.

Example:


<figure>
	<img src="example.jpg" alt="Example Image">
	<figcaption>This is an example image.</figcaption>
</figure>

<figure>

Definition: The <figure> element is used to encapsulate media content, such as images, diagrams, or videos, along with an optional caption using <figcaption>.

Example:


<figure>
	<img src="example.jpg" alt="An example image">
	<figcaption>An image showing an example.</figcaption>
</figure>

<footer>

Definition: The <footer> element defines the footer for a document or section. It typically contains information about the author, copyright, or related links.

Example:


<footer>
	<p>Copyright © 2024 My Website</footer>

<header>

Definition: The <header> element represents introductory content for a document or section. It can include a site logo, navigation, or a heading.

Example:


<header>
	<h1>Welcome to My Website</h1>
	<nav>Links to sections</nav>
</header>

<main>

Definition: The <main> element represents the dominant content of the body of a document. It should be unique and not repeat headers, footers, or navigation.

Example:


<main>

7.1 Introduction to Forms

Forms are an essential part of web development, allowing users to input data that can be sent to a server. In HTML, the <form> element is used to define a form, which can contain input fields, buttons, checkboxes, and other elements.

7.1.1 What Are Forms?

Definition:
A form in HTML is a container for different types of input elements, such as text fields, buttons, and checkboxes. Forms allow users to interact with the web page and submit data, which can then be processed on the server.

Example:


<form action="submit_form.php" method="post">
	<label for="name">Name:</label>
	<input type="text" id="name" name="name">
	<input type="submit" value="Submit">
</form>

Output:

7.2 Form Attributes

The <form> element has several important attributes that define its behavior, such as action, method, and target.

7.2.1 Common Form Attributes

  • action: Specifies the URL where the form data will be sent when submitted. It can be a relative or absolute URL.
  • method: Defines how the form data will be sent to the server. Common methods are GET and POST.
  • target: Specifies where to display the response after submitting the form. Common values are _self, _blank, etc.

Example: A form with action and method attributes:


<form action="submit_form.php" method="post">
	<label for="email">Email:</label>
	<input type="email" id="email" name="email">
	<input type="submit" value="Submit">
</form>

Output:

7.3 Form Elements

Form elements are the components of a form, allowing users to input various types of data. These elements can include text fields, checkboxes, radio buttons, and more.

7.3.1 Common Form Elements

  • <input>: Defines various types of input fields, such as text, password, and radio buttons.
  • <textarea>: Allows for multi-line text input, such as comments or messages.
  • <select>: Provides a dropdown menu for selecting a value from a list.
  • <button>: Defines a clickable button that can submit or reset a form.

Example: A form with text, radio, submit button, and a textarea:


<form>
	<label for="username">Username:</label>
	<input type="text" id="username" name="username">
	<br>
	<label for="gender">Gender:</label>
	<input type="radio" id="male" name="gender" value="male"> Male
	<input type="radio" id="female" name="gender" value="female"> Female
	<br>
	<label for="message">Message:</label>
	<textarea id="message" name="message" rows="4" cols="50"></textarea>
	<br>
	<input type="submit" value="Submit">
</form>

Output:


Male Female

8.1 Introduction to Media

Media elements, such as video and audio, allow us to embed multimedia content on web pages. The <video> and <audio> elements provide native support for embedding media, with attributes that help control their behavior.

8.1.1 What Are Media Elements?

Definition:
The <video> and <audio> elements allow you to embed video and audio content into a web page. They can have attributes that control playback, volume, looping, and more.

8.1.2 Why Use Media Elements?

Media elements allow for easy integration of video and audio without relying on external plugins. These elements are natively supported by modern browsers, making media playback smoother and more accessible.

8.2 Video Element

The <video> element is used to embed video files in a web page. It can include several attributes to control its behavior.

8.2.1 Common Video Attributes

  • src: Specifies the path to the video file. Can be a URL or a relative path.
  • controls: Adds playback controls like play, pause, and volume control to the video.
  • autoplay: Automatically starts playing the video as soon as it is ready.
  • loop: Makes the video loop continuously once it reaches the end.
  • muted: Mutes the video’s audio by default when it is loaded.
  • poster: Specifies an image to display before the video starts playing.
  • width: Defines the width of the video player in pixels.
  • height: Defines the height of the video player in pixels.
  • preload: Specifies how the video should be preloaded, with options like auto, metadata, or none.

8.2.2 Example of a Video

Example:


<video src="video.mp4" controls autoplay loop muted poster="poster.jpg" 
width="600" height="400">
	Your browser does not support the video element.
</video>

Output:

8.3 Audio Element

The <audio> element is used to embed audio files in a web page. It can also include various attributes for controlling playback and behavior.

8.3.1 Common Audio Attributes

  • src: Specifies the path to the audio file. Can be a URL or a relative path.
  • controls: Adds playback controls such as play, pause, and volume control to the audio.
  • autoplay: Automatically starts playing the audio as soon as it is ready.
  • loop: Makes the audio play continuously once it reaches the end.
  • muted: Mutes the audio by default when it is loaded.
  • preload: Specifies how the audio should be preloaded, with options like auto, metadata, or none.
  • volume: Sets the initial volume level of the audio (ranges from 0 to 1).

8.3.2 Example of an Audio

Example:


<audio src="audio.mp3" controls autoplay loop muted preload="auto">
	Your browser does not support the audio element.
</audio>

9.1 HTML Metadata (Meta Tags)

Metadata in HTML provides information about the web page, such as its description, keywords, author, and more. It is placed inside the <head> section of the HTML document and is used to control how the page appears on search engines and social media.

9.1.1 Common Meta Tags

  • <meta charset="UTF-8">: Defines the character encoding for the document.
  • <meta name="description" content="Web page description">: Provides a short description of the page for search engines.
  • <meta name="keywords" content="HTML, CSS, JavaScript">: Provides search engine keywords.
  • <meta name="author" content="Your Name">: Specifies the author of the web page.
  • <meta http-equiv="refresh" content="30">: Refreshes the page every 30 seconds.

9.2 HTML Accessibility

HTML accessibility ensures that your website is usable by people with disabilities. It involves providing alternative text for images, proper heading structures, and ARIA (Accessible Rich Internet Applications) roles.

9.2.1 Key Concepts

  • Alt Text: Descriptive text for images to help screen readers.
  • ARIA Roles: Provide roles to elements for accessibility (e.g., role="button").
  • Semantic HTML: Using proper HTML tags for content, like <header>, <nav>, <article>, etc.

9.3 HTML Data Attributes

Data attributes allow you to store extra data in HTML elements. They are used to store custom information that can be accessed using JavaScript.

9.3.1 Example

<div data-user="123" data-role="admin">User Info</div>

In JavaScript:

let userData = document.querySelector('div').dataset.user; // 123

9.4 HTML Global Attributes

Global attributes are attributes that can be used on any HTML element. They provide common functionalities like event handling, styling, and accessibility.

9.4.1 Common Global Attributes

  • id: Identifies the element uniquely.
  • class: Specifies a class for CSS styling.
  • style: Defines inline CSS styles.
  • title: Provides additional information when the user hovers over an element.
  • lang: Specifies the language of the element’s content.

9.5 HTML Navigation (<nav>)
